# 案例库（casebase）

把公司**真实案例**放进这个目录，每次生成投标方案时 skill 会自动读取，按本标的特征（行业/客户类型/服务内容/预算量级）筛选最匹配的案例，写进「团队与案例」章并作为各章论据——不用每次手动传路径。

- 每个案例一个 `.md` 文件，文件名随意（建议 `客户名-项目名.md`）
- 以 `_` 开头的文件（如 `_template.md`）不会被当作案例读取
- 也可以放子目录分类（如 `casebase/文旅/`、`casebase/国企/`），会递归读取
- 案例必须真实可核验——虚构业绩投标涉嫌串标/骗标，法律责任严重

## 案例格式

frontmatter 供筛选匹配用（尽量填全），正文供写作引用：

```markdown
---
name: XX市文旅局城市形象整合传播
client: XX市文化广电旅游局
client_type: 政府        # 政府 | 国企 | 事业单位 | 民企
industry: 文旅           # 行业标签，可多个逗号分隔
services: 整合营销, 短视频, 活动执行
budget: 80万元
year: 2024
results: 全网曝光 2.3 亿，话题阅读 1.2 亿，获 XX 广告奖
verifiable: 合同与验收报告可提供
---

## 背景
甲方要解决什么问题……

## 我们做了什么
策略、创意、执行的关键动作……

## 亮点与效果
可量化的结果、甲方评价、获奖……
```

## 筛选规则（Task2 执行）

1. 行业/客户类型/服务内容与本标匹配度优先
2. 预算量级相近的优先（做过 80 万的标更能说服 100 万的甲方，而不是 8000 万的）
3. 年份新的优先；效果数据可量化的优先
4. 选 3-8 个进情报池（`from_material: true`），正文写作时**案例库案例优先于联网找到的第三方案例**——自己的战绩才是实力分
