# Task 2b — 综述章

你在其余章全部定稿后，串行写综述/摘要章。综述是合成物，只压缩已有正文，不新增主张或承诺。

占位变量：`{LANG}` · `{SUMMARY_BRIEF}`（一页纸 + 全部定稿章的标题与核心判断）· `{TMPDIR}`。

## 交付目标
用一章篇幅让甲方在读正文前就抓住主张与方案骨架，且每一句都能在后文找到出处。

## 依据
- `{SUMMARY_BRIEF}`：一页纸、各章标题与核心判断。
- 已定稿的 `{TMPDIR}/_sections/section-*.md`。
- `references/writing-patterns.md`。

## 工作顺序
1. 复述一页纸主张与记忆句作为综述主线。
2. 按章序各用一两句给出该章的客户判断与最强证据，指向后文。
3. 收束到 signature 主亮点。

## 输出契约
- `{TMPDIR}/_sections/section-0.md`（综述，放全案最前）。
- 返回 JSON：`{"section_n": 0, "chars": N, "new_claims": []}`。

## 完成判据
- 综述不含任何后文没有的主张、数字或承诺（`new_claims` 必须为空）。
- 主张与记忆句与一页纸完全一致（记忆句逐字）。
- 用综述自己的话压缩，**不逐字复制某章的开头或原句**。
