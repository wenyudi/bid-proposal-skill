# 校准策略品味

本指南把你自己的赢标经验和真实失败反馈转成可复用的策略先验。它不要求修改 Python，也不会把客户材料自动加入公共规则。

## 1. 准备 3–5 份可用样本

优先选择你已确认中标、且能获得客户/评委反馈或内部复盘的方案。先去除客户名称、个人信息、金额、关系信息和未授权案例；如果无法安全脱敏，只在仓库外复盘。

不要把“中标”直接等同于“文案好”。同时记录价格、资质、合规、既有合作等已知变量，避免把其他因素误归因给某个句式。

## 2. 只抽策略骨架

按 [一页纸策略骨架](../../references/strategy-patterns.md) 的模板，逐案提取：

- 客户张力与依据；
- 尖锐洞察、核心命题和实际被复述的记忆句；
- 当时最强替代命题、最终选择的决定性依据和接受的取舍；
- 洞察→策略→表达→执行→证明；
- 最能代表主张的 signature 成果，以及客户是否真正记住或使用它；
- 每章的独有贡献；
- 只有本投标人能成立的真实机制或证据；
- 客户实际表扬、追问或否决的内容。

至少三份样本重复出现且有反馈支持的结构，才考虑提升为通用先验。其余保留为场景特例。

## 3. 记录一次不满意输出

复制 [`_quality-lessons/_template.md`](../../casebase/_quality-lessons/_template.md)，一次只选一个主失败：

- `strategy_hollow`：洞察或主张空洞，名称互换仍成立；
- `throughline_break`：一页纸成立，但章节没有共同推导；
- `cliche_style`：策略基本正确，正文被套话覆盖。

保存原始 exact quote、为什么失效、改后版本和真实反馈。不要只写“感觉不好”，也不要为了积累样本编造客户评价。

## 4. 提升为共享对照

只有人工确认有效的记录才能提升：

- 策略判断差异放入 [`contrast-examples.md`](../../references/contrast-examples.md)；
- 反复出现的具体坏句和诊断放入 [`anti-patterns.md`](../../references/anti-patterns.md)；
- 跨 3–5 份赢标案稳定出现的骨架才更新 [`strategy-patterns.md`](../../references/strategy-patterns.md)。
- 经多次验证、能直接指导正文构造的正向动作更新 [`writing-patterns.md`](../../references/writing-patterns.md)。

提升时保留“合成/脱敏”标记和适用边界。术语不做机械禁词；真正要沉淀的是为什么失效、什么机制使改稿更好。`anti-patterns.md` 服务独立批评与复盘，writer 的生成上下文只接收已经验证的正向动作。

## 5. 验证没有过拟合

用一份不同标型的虚构 fixture 做 shadow run，检查：

- candidate 池没有因 rubric 提前收窄；
- 一页纸能明确说明政府/公共、商业或 hybrid 的主判断标准；
- 章节 brief 均携带同一记忆句和 section spine；
- 红队能精确引用通用段落，而不是按词表误杀；
- 新先验没有引入客户私密事实、未授权案例或不可兑现承诺。

如果只改善某一类项目，把经验保留为场景模板，不要升级为全局规则。
