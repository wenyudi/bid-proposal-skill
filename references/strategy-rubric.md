# 策略质量 Rubric

本 rubric 定义质量上限，不计算总分，也不替代人工判断。Task 2.5 逐维给出 `level + finding + next_revision`；独立策略批评者重新判断，不能照抄自评。

等级只用 `deficient / fragile / adequate / strong / distinctive`：

- `deficient`：没有形成可用判断，或与项目事实冲突。
- `fragile`：方向可见，但主要靠行业常识、口号或未证实跳步。
- `adequate`：项目相关、逻辑成立，可安全进入讨论。
- `strong`：选择清晰、推导完整，能影响评委判断和后续写作。
- `distinctive`：除 strong 外，还具有难替换的项目特异性和投标人可信优势。

## 五个维度

### 洞察锐度 `insight_sharpness`

- deficient：复述标书或客户目标。
- fragile：说出行业痛点，但没有本项目张力或依据。
- adequate：指出一个有据的项目断点，并解释为什么重要。
- strong：揭示表面需求背后的因果或取舍，能改变方案方向。
- distinctive：洞察既反直觉又可辩护，并使常规解法显得不够。

检查：删掉客户名后，它是否仍只是“传播不够、协同不足、需要提升影响力”？若是，尚不是洞察。

### 可复述性 `recallability`

- deficient：没有一句核心主张。
- fragile：句子顺口但空泛，或同时塞入三件事。
- adequate：一句话能说明客户结果与关键机制。
- strong：评委能在十秒内复述，并用它解释主要章节。
- distinctive：主张还能自然成为比较竞争方案的判断标准。

检查：合上 80 页方案，只允许复述一句；若需要列三条“主题”，主张尚未收敛。

### 推导闭环 `deductive_coherence`

- deficient：策略、创意、执行并列堆砌或互相矛盾。
- fragile：有主线词，但章节只是在重复它。
- adequate：洞察 → 策略 → 表达 → 执行 → 证明基本成立。
- strong：每一步都是上一判断的结果，每章都有不可替代的贡献和交接。
- distinctive：删掉任一步都会明显破坏说服链，且摘要没有强于正文。

检查：逐步追问“所以呢”；若下一步也可从任何洞察推出，就是跳步。

### 差异化 `differentiation`

- deficient：竞争对手名称互换后完全成立。
- fragile：只有形容词、服务态度、流程完整或未经证明的资源优势。
- adequate：至少有一个项目特定选择或机制，互换测试只部分成立。
- strong：主张、机制与证据形成难替换组合，名称互换失败。
- distinctive：客户可据此建立选择标准，竞争对手只能改变其方案而非换名跟随。

互换测试：把我方名称换成主要竞争对手。`passes` 表示整段仍成立，是失败信号；`fails` 表示因真实能力、机制或项目选择而不再成立。

### 落地可信度 `delivery_credibility`

- deficient：只有愿景、活动名或流程名词。
- fragile：有动作，没有责任、资源、时点、风险或验收闭环。
- adequate：关键动作可执行，边界诚实，评委知道如何检查。
- strong：创意与交付是一条推导链，客户管理负担和失败回退均被处理。
- distinctive：可见成果让评委在正文中提前体验交付质量，而非等附件或口头承诺。

## 策略关卡

人工只回答一次：“是否按这一页进入写作？”批准的是方向与取舍，不是每个句子。`approved` 才可直接递交；`assumed` 只允许 `-auto` 或迁移后的安全草案。rubric 的弱项必须展示，但不能靠伪精确总分替人做决定。
