# 客户正文正向写作模式

本文件服务 Task 2 正文写作。它提供可组合的写作动作与底线，不提供项目事实、固定句式或章节模板；具体内容来自当前章节 brief。

## 开篇与标题
- 每章**以客户任务或结果开篇**：先用一句客户判断说明本章解决什么、为何影响选择；不以"我司成立于…""我们拥有…"这类自夸开篇。
- **标题表达判断、含主张**：标题给结论，正文完成推导；子节标题也表达该节要点，而非贴标签。
- 我方能力通过机制、责任和成果自然显现，而不是形容词。

## 一章的说服任务
开篇之后依次交付：
1. 项目依据：来自标书要求、客户需求或素材的具体事实；
2. 方案选择：本章的关键取舍及其与核心主张的关系；
3. 实现机制：动作、责任、时点、资源边界、协作接口和回退路径；
4. 可见成果：评委此刻就能检查的表格、清单、样例、分镜、节奏表或看板；
5. 判断交接：本章证明了什么、下一章在此基础上继续证明什么。

每章形成一个独有认知增量；记忆句统一方向，章节内容逐步增加理由与证据。

## 段落构造
按内容需要组合正向构造：
- 判断 → 依据 → 含义
- 客户摩擦 → 方案动作 → 可感知变化
- 动作 → 责任 → 时点 → 检查
- 目标 → 口径 → 数据源 → 复盘动作
- 风险触发 → 安全回退 → 升级路径
- 拟议样例 → 字段实值 → 适用边界

一个段落优先完成一个评委判断；主张后紧跟依据/机制/成果，段落结尾落到客户结果或检查方式。

## 客户可见成果
把要评委检查的东西直接写成正常方案内容：矩阵展示精确对应，清单展示责任/时点/完成标准，样例让评委提前体验交付，节奏表展示阶段与检查点，看板展示指标口径/数据源/复盘。全案只把一个 signature 主亮点放在最清晰的版面，并先交代它替评委解决的判断；其余成果作支撑与查验，让"完整"与"重点突出"同时成立。版式服从信息关系与阅读效率。

## 语言质感
- 以具体对象和动作动词承载句子（"核对入口有效期""在里程碑签收"）。
- 用客户结果开启或收束关键段落。
- 专业术语紧邻其对象、作用和检查方式，让术语承担信息而非气氛。
- 长短句交替；重复信息用表格或交叉引用压缩。
- 政府/公共项目突出正当性、连续性、可审计性与协同负担；商业项目突出行为摩擦、生意机制、验证口径与资产沉淀。

## 套话重写
"赋能、生态、闭环、矩阵、全域、多维、抓手、引擎、焕新、聚势"等不是机械禁词。删掉术语后，同句或紧邻内容若找不到**具体对象、变化机制、项目依据、责任/时点、可检查结果**中的多数，**整句删除重写**，而非替换同义词。具体好坏对照见 `references/contrast-examples.md`。

## 真实性表达
- 事实用素材里的具体投影；拟议动作用方案语气；illustrative 内容写清是本项目拟议样例。
- 缺失素材用**具体、可信**的内容虚构补全，让方案成熟完整——但每一处虚构/假设都要进本章 `fabrications` 申报（位置 / 虚构了什么 / 应核实或替换 / 风险等级）。不留占位符、不写空话。
- 交付前从客户视角复述三件事：本章新判断、支持判断的最强依据、客户能当场检查的成果。三者清楚且相连即可交付复核。

## 客户稿编号
| 层级 | Markdown | 格式 | 示例 |
|:---|:---|:---|:---|
| 章 | `##` | 汉字数字 + 顿号 | `## 三、传播策略与大概念` |
| 子节 | `###` | 阿拉伯数字 N.x | `### 3.1 核心传播主张` |
| 子子节 | `####` | 括号阿拉伯数字 | `#### (1) 线上引爆路径` |

同层统一、不可跳级；每章子节从 N.1 开始。内部标签不进标题。

## 递交稿禁项
以下一律不出现在客户正文：
- URL / 书目；内部 ID；模型或工具痕迹；叙事策略、模式、版本、生成时间等内部痕迹。
- 私密原句、未确认的个人偏好、内部竞争情报。
- 销售 CTA、"期待与您沟通/签约"、分档 opt-in 报价、"排除项/不包含"式责任转嫁。
- 把拟议偷写成"确保/保证"、无限责任、未授权的 KPI/SLA/赔偿/免费资源。
- LaTeX；货币 `$` 未转义；章节编号混层；占位符被当成最终事实。
