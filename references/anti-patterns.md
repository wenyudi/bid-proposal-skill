# 反模式与失败归因

本文件不是机械禁词表。术语只有在删掉后仍留下对象、机制、证据与结果时才有意义。命中一个词不自动判错；整句经不起追问才重写。

## 具体坏句

| 坏句 | 根因 | 必须回答的重写问题 |
|:---|:---|:---|
| “以创新驱动赋能项目高质量发展。” | 没有对象和因果 | 谁发生什么变化？靠哪一动作？ |
| “构建多维传播矩阵，形成全域触达。” | 渠道清单冒充策略 | 每个渠道承担哪一个不同决策任务？ |
| “打造内容生产、传播、反馈的闭环。” | 流程名词冒充机制 | 反馈由谁在何时改变下一轮什么？ |
| “全面提升品牌影响力和美誉度。” | 结果不可检查 | 针对谁、改变何种判断、用什么口径观察？ |
| “整合优质资源，实现 1+1>2。” | 未证明的资源与数学修辞 | 哪些已确认资源以什么组合产生增量？ |
| “依托专业团队和丰富经验保驾护航。” | 公司自夸且可互换 | 哪个已核验角色承担哪项责任和验收？ |
| “通过线上线下一体化提升用户体验。” | 没有真实摩擦点 | 用户在哪一步受阻，整合后少做什么？ |
| “建立长效机制，沉淀数字资产。” | 未来名词替代交付 | 本合同期交出什么、归谁、如何复用和验收？ |

高风险套话包括“赋能、生态、闭环、矩阵、全域、多维、抓手、引擎、焕新、聚势”。它们可以出现，但同一句或紧邻内容必须能找到：具体对象、变化机制、项目依据、责任/时点以及可检查结果。缺两项以上，优先整句删除而非替换同义词。

## 三类主失败

### `strategy_hollow` 策略空洞

症状：洞察复述标书；主张可用于任何客户；互换测试通过；章节再漂亮也没有选择。回到 Task 2.5 的一页纸策略，不让 writer 润色掩盖。

### `throughline_break` 主线断裂

症状：一页纸成立，但某章无法说明如何支撑主张；策略、创意、执行变成三组并列清单；摘要强于正文。修复 Section 的 `strategy_role` 与交接，再重编译受快照绑定的正文。

### `cliche_style` 文风套话

症状：底层逻辑基本正确，但正文被“赋能、矩阵、闭环、全面提升”覆盖；删掉术语后没有实物。由 Task 3 对 exact quote 重写，不改 canonical 语义。

一次不满意只选一个主失败，可附次级标签。先修最上游根因，再判断下游问题是否自然消失。

## 将失败变成品味资产

每次真实使用后保存：场景、原始坏句、一种主失败、为何失效、改后版本、客户/评委反馈、是否可推广。没有反馈的改写只能算候选教训；经人工确认后，才把短小对照提升进本文件或 `contrast-examples.md`。不要把客户私密原文、未授权案例或一次中标结果直接固化为通用规则。
