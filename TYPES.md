# 提案类型与章节结构 — proposal

面向广告/传媒公司给**政企客户**投标的方案。基于《政府采购法》《招标投标法》评标办法、常见政企传媒采购评分办法、4A/本土代理商提案（Pitch）方法论整理。

> 用法：Task 1 从标书中判定最匹配的一类，套用其「典型章序」作为方案骨架的起点，再按标书的**评分办法**逐项校准增删。类型是起点不是牢笼——整合型标书可跨类型融合。

---

## 类型一：品牌传播 / 整合营销（IMC）

| 项目 | 说明 |
|:----|:-----|
| 适用场景 | 品牌年度传播、形象升级、整合营销战役 |
| 典型章序 | 项目理解与洞察 → 传播策略与大概念 → 内容与创意 → 媒介与渠道组合 → 传播节奏与排期 → 效果目标与评估 → 报价与资源 → 团队与案例 → 风险与保障 |
| 评委最看重 | 策略逻辑是否闭环、创意大概念、媒介组合的性价比 |
| 惊喜发力点 | 一个可被记住的 Big Idea + 一个低成本高传播的引爆点 |

## 类型二：活动 / 事件营销（Activation / Event）

| 项目 | 说明 |
|:----|:-----|
| 适用场景 | 发布会、节庆活动、路演、展会、快闪、赛事 |
| 典型章序 | 活动理解与目标 → 主题创意与视觉 → 活动流程与体验设计 → 现场执行与安全预案 → 传播引流（前中后） → 排期与分工 → 报价与资源 → 团队与案例 → 风险与应急 |
| 评委最看重 | 现场落地能力、执行细节、应急预案、体验亮点 |
| 惊喜发力点 | 一个刷屏级互动/装置 + 一套"零事故"执行保障 |

## 类型三：数字营销 / 新媒体运营（Digital / Social）

| 项目 | 说明 |
|:----|:-----|
| 适用场景 | 政务新媒体代运营、社媒矩阵、私域、内容营销、投放代运营 |
| 典型章序 | 现状诊断与目标 → 内容策略与栏目规划 → 平台矩阵与分发 → 投放与增长策略 → 数据运营与工具 → 排期与更新机制 → 报价与资源 → 团队与案例 → 风险与合规 |
| 评委最看重 | 内容可持续产出能力、数据驱动、平台规则熟悉度、合规（政务尤重） |
| 惊喜发力点 | 一个爆款内容方法论 + 一套可视化数据看板 |

## 类型四：政务宣传 / 城市形象（Gov Publicity / City Branding）

| 项目 | 说明 |
|:----|:-----|
| 适用场景 | 城市/文旅形象、政策解读传播、主题宣传、公益倡导 |
| 典型章序 | 政策与舆情理解 → 传播定位与主线 → 内容矩阵与创意 → 主流媒体+新媒体组合 → 落地活动 → 舆情引导与风险 → 排期 → 报价与资源 → 团队与案例 |
| 评委最看重 | 政治正确与导向把握、主流媒体资源、正能量传播、舆情安全 |
| 惊喜发力点 | 一个有温度的城市叙事切口 + 权威媒体背书组合 |

## 类型五：影视 / 内容制作（Video / Content Production）

| 项目 | 说明 |
|:----|:-----|
| 适用场景 | 宣传片、纪录片、TVC、短视频系列、微电影 |
| 典型章序 | 创作理解 → 主题立意与叙事 → 脚本/分镜方向 → 视觉与制作方案 → 制作流程与团队 → 排期与交付标准 → 报价与资源 → 案例与样片 → 风险与版权保障 |
| 评委最看重 | 创作立意、制作水准（样片）、交付确定性、版权合规 |
| 惊喜发力点 | 一个新颖叙事视角 + 明确的成片质感承诺（附样片对标） |

## 类型六：媒介采购 / 投放（Media Buying / Placement）

| 项目 | 说明 |
|:----|:-----|
| 适用场景 | 户外/线上/电视媒介采购、广告投放执行 |
| 典型章序 | 投放目标理解 → 媒介策略与选择逻辑 → 排期与预算分配 → 采购资源与价格优势 → 监播与优化机制 → 效果评估 → 报价明细 → 团队与资源 → 风险与保障 |
| 评委最看重 | 媒介性价比、资源真实性、监播与结案能力、报价明细 |
| 惊喜发力点 | 独家/优价媒介资源 + 透明可核验的监播结案体系 |

## v3 按标型加载的可覆盖先验

以下只用于 Task 1 的**完备性检查与研究排序**，不是事实、固定权重或章节模板。默认置信度最多 medium；标书、澄清、公开组织职责和用户确认随时覆盖。先验不得凭空生成正式权力、否决权、个人偏好、selected VP、committed Claim 或 submission blocker。

| 标型 | 优先检查的角色原型 | 常见判断主进程 | 优先补的 Evidence / Delivery burden |
|:---|:---|:---|:---|
| 品牌传播 / IMC | 业务负责人、决策领导、评标专家、采购合规 | 理解品牌任务 → 相信洞察/定位 → 认可 Big Idea 与组合价值 → 相信跨渠道落地 → 确认导向/预算/版权 → 可辩护选择 | 受众与品牌现状依据、创意机制、渠道协同、同类案例可迁移边界、效果口径、内容版权和舆情预案 |
| 活动 / 事件营销 | 业务负责人、采购合规、最终使用者/参与者、财务纪检 | 理解活动目标 → 认可体验价值 → 相信流程/人员/场地能执行 → 确认安全与应急 → 选择可控方案 | run-of-show、场地/供应商/人员容量、客流与安全依据、审批窗口、彩排、应急 fallback、验收与留痕 |
| 数字营销 / 新媒体运营 | 业务负责人、最终使用者、评标专家、采购合规 | 相信诊断 → 认可内容/增长机制 → 相信持续产能与数据闭环 → 确认平台/政务合规 → 选择可持续运营 | 客户/平台基线、栏目与更新产能、人员排班、平台规则、数据源/看板/优化频率、舆情与账号安全 |
| 政务宣传 / 城市形象 | 决策领导、业务负责人、采购合规、最终公众/使用者 | 理解战略与导向 → 相信城市/政策叙事 → 认可公共价值与传播体验 → 确认权威性/舆情安全 → 相信落地 → 选择稳妥且有记忆点的方案 | 政策/规划原文、公开城市事实、受众与舆情依据、主流媒体/内容资源真实性、导向审校、危机升级机制 |
| 影视 / 内容制作 | 业务负责人、决策领导、评标专家、采购合规 | 理解创作任务 → 认可立意/质感 → 相信样片与制作方法 → 相信里程碑交付 → 确认版权/审改/成片验收 → 选择 | 样片与我方角色、主创可用性、脚本/分镜/样片确认、设备与制作排期、素材授权、版本审改与成片验收 rubric |
| 媒介采购 / 投放 | 采购合规、财务纪检、业务负责人、评标专家 | 理解投放目标 → 相信媒介选择与测算 → 认可投入产出/资源优势 → 相信采购与优化执行 → 确认资源真实性/监播/审计 → 选择 | 刊例与授权、库存/点位真实性、价格与折扣依据、覆盖频次测算、排期容量、监播证据、结案与异常补量规则 |

### 先验使用规则

1. Task 1 仍先读本标的 Requirement、角色依据和组织职责；表中未列角色不等于不重要，列出的角色也可标 not_applicable 并说明理由。
2. `job_kind` 仍只有 understand/believe/value/deliver/safe/choose；表中顺序可回访、合并或重排，评分骨架永远优先，不为补齐旅程新增孤儿章。
3. 六标型共用十个 customer-fit 维度；只由本标 scoring 映射、正式权力、履约/审查风险和 selected 组合动态调节 weight range，不建立按类型固定总权重。
4. 类型先验只能生成 research gap、candidate Role/Need/VP/Action 或低置信度提醒；不能单独升级 required path、Evidence 事实或硬承诺。
5. 真实标使用后，把误报、遗漏、角色断点、Evidence 负担和客户反馈记录为校准样本；只在同类样本反复出现且可解释时调整先验，不因单次胜负固化偏见。

## 评标维度模型（评分办法通常由这几类构成）

政企传媒标的评分办法多为「技术分 + 商务分 + 价格分」结构。Task 1 拆标书时，把每个评分项归入下列维度，并记录**权重**——权重高的维度就是方案投入重心。

| 维度 | 常见权重带 | 评分项示例 | 方案对应章节 |
|:-----|:----------|:----------|:------------|
| **技术/服务方案** | 30-50% | 整体策划思路、内容方案、执行方案、进度计划 | 洞察/策略/创意/执行/排期章 |
| **创意/策划** | 10-25% | 创意水平、主题契合度、亮点 | 大概念/创意章 + 差异化增值点 |
| **企业实力/资质** | 10-20% | 注册资本、资质证书、类似业绩 | 团队与案例章 + 资格响应 |
| **团队配置** | 5-15% | 项目组人员、职称、经验 | 团队章 |
| **报价/价格** | 10-30% | 报价合理性、明细完整性 | 报价与资源配置章 |
| **服务保障** | 5-15% | 售后、应急、质量承诺 | 风险与保障章 |

> 判定原则：**权重决定篇幅与投入**。技术/服务方案通常权重最高 → 方案主体；价格分是低价竞争项 → 用"最优解"讲清性价比而非单纯压价；创意分是拉开差距的关键 → 惊喜点集中发力处。

### 评委画像（谁在打分，各自在意什么）

| 评委类型 | 在意 | 打动方式 |
|:--------|:-----|:--------|
| **采购人代表（业务口）** | 能不能解决我的实际问题、KPI、省不省心 | 甲方视角开篇、可落地、有专人对接 |
| **技术/行业专家** | 专业性、逻辑闭环、创意水准、可行性 | 方法论、数据、样片/案例、执行细节 |
| **财务/纪检** | 报价合规、无违规、资质真实 | 报价明细、合规响应、真实业绩 |

## 红队四视角（Task 4 定稿前评审）

方案写完不等于能中。定稿前派 4 个 agent 并行攻击，模拟真实评标现场。前三个是评委画像的攻击态，第四个是你在评标室里看不见的那个人。

> 用法：Task4 读 `prompts/task4_redteam.md`，替换 `{ROLE_KEY}` `{ROLE_NAME}` `{ROLE_BRIEF}` 后并行派发 4 次。产出汇总到人工定稿关卡（Gate 2），按 DECISIONS.md **一次处理一条质疑**——红队只提质疑，不自动改稿。

### R1 `buyer` — 采购人代表（业务口）

> 你是这个项目的实际使用方，中标后要跟这家公司天天打交道，做砸了是你担责。你不懂广告行话，只关心：这方案解决我的问题吗？他们真懂我们单位吗？出了事谁负责？会不会又要我天天盯着？
> **攻击重点**：自嗨段落、不接地气的创意、没有专人对接机制、对我方业务的理解流于表面、承诺听着漂亮但没人兜底。
> **口头禅**："这段在讲你们自己，跟我有什么关系？"

### R2 `expert` — 技术/行业专家

> 你是评标专家库随机抽来的资深从业者，见过太多花架子。你在意专业性、逻辑闭环、方法论是否成立、创意能否落地、数据是否经得起推敲。
> **攻击重点**：论证链断裂、方法论套模板、创意无执行路径/资源/排期、数据无来源或来源不可靠、案例与本项目不可比、KPI 无测算依据。
> **口头禅**："这个数是怎么算出来的？这个案例跟本项目可比吗？"

### R3 `audit` — 财务/纪检

> 你不看创意，只看合规和钱。你要确保这个标经得起审计和投诉。
> **攻击重点**：报价超预算或与内容不匹配、分项金额不合理、资质业绩无法核验、实质性条款负偏离、"排除项"式免责措辞、承诺超出能力范围、引用了私下沟通（不正当接触嫌疑）、政务导向问题。
> **口头禅**："这条能核验吗？出了问题谁负责？"

### R4 `rival` — 竞争对手

> 你是同台竞标的另一家代理商，你拿到了这份方案。你的任务是在评委面前把它比下去。你知道行业里谁有什么资源、什么话是吹的、哪些承诺是空头支票。
> **攻击重点**：找出对手可以轻易复制的"差异化"、找出虚高的承诺、找出可以用更低价或更强资源碾压的环节、找出方案里最脆弱的一环并放大。
> **口头禅**："这个所谓的差异化，我们也能做，而且更便宜。"

> **红队不改稿。** 它的产出是「质疑清单」，交给人在 Gate 2 逐条决定：哪些必须补、哪些是红队想多了、哪些暴露了真实的能力缺口需要调整投标策略甚至弃标。

## 叙事策略库（Narrative Strategy）

同样的内容，讲法不同，评委的体感完全不同。Task 1 制定策略时必须选定**一个主叙事**（可配一个辅助叙事），写入 `strategy.json` 的 `narrative` 字段，贯穿全案；Task 3 各章按注入的叙事指令写作。

> **铁律：叙事是讲法，不是内容裁剪。** 无论选哪种叙事，评分点必须全部答到、创意必须配落地、报价必须讲清价值——叙事只决定"怎么组织、怎么开篇、用什么语言温度"，不允许以"故事讲不下"为由跳过任何评分项。

### N1 `logic` — 逻辑征服

| 项目 | 说明 |
|:----|:-----|
| 一句话内核 | 用无懈可击的论证链让评委"不得不服"：问题 → 归因 → 方法论 → 路径 → 证据，环环相扣 |
| 适用信号 | 技术/服务方案分权重 ≥40%；评委以行业/技术专家为主；暗标（不见人只见文）；机制复杂的标（媒介采购/数字运营/数据服务） |
| 章内写法 | 开篇给"对问题本质的判断"；每个主张后立刻跟"依据 + 推演"；多用结构化表达（对比表/决策依据/路径拆解）；收尾回扣论证主链 |
| 语言温度 | 冷静、精确、克制；形容词少，动词和数字多 |
| 风险禁忌 | 忌写成枯燥说明书——每章仍要有一个"认知增量"（评委没想到的洞察）；论证链断一环，全案失信 |

### N2 `story` — 故事打动

| 项目 | 说明 |
|:----|:-----|
| 一句话内核 | 把方案讲成一个有主人公的故事：主人公是甲方/城市/用户（绝不是我们），冲突是痛点，转机是方案，结局是可感知的改变 |
| 适用信号 | 城市形象/文旅/公益/品牌传播类；"创意水平/主题契合"评分权重高；甲方近期传播有情感化叙事倾向；明标有讲标/答辩环节 |
| 章内写法 | 开篇用场景/人物/细节把评委带入；按叙事弧线组织（现状之待变 → 转机出现 → 图景展开）；创意用"这一幕会发生什么"呈现；数据作为故事的锚点而非主角；每个场景后必须落回"怎么做" |
| 语言温度 | 有画面感、有温度，但克制不煽情 |
| 风险禁忌 | 政务标必须导向正确：现状要写成"发展的下一步"，不渲染负面（"问题很严重"是雷区）；故事线不得盖过评分点应答；场景素材优先用 Task2 查到的真实细节，不虚构具体人物事迹 |

### N3 `vision` — 愿景共创

| 项目 | 说明 |
|:----|:-----|
| 一句话内核 | 画一张甲方置身其中的未来图景，把投标关系升维成伙伴关系："三年后的XX将是……而我们是同行者" |
| 适用信号 | 多年框架/长期代运营标；标书里有明确战略语境（发展规划/城市定位/上级要求）；评分办法重"服务规划/合作机制/可持续性" |
| 章内写法 | 开篇从甲方的战略语境切入（引其规划表述）；方案呈现为里程碑路线图（首期立住 → 中期放大 → 远期沉淀）；强调共建机制（联合工作组/知识沉淀/能力移交） |
| 语言温度 | 开阔、笃定、有担当 |
| 风险禁忌 | 忌空喊愿景无路径——每个愿景节点必须配当期可交付物；首期（本合同期）的落地必须最重最实，否则评委只看到画饼 |

### N4 `evidence` — 数据实证

| 项目 | 说明 |
|:----|:-----|
| 一句话内核 | 用数据、对标和可核验的结果说话："同类项目做到过 X，本项目路径一致，结果可期" |
| 适用信号 | 效果导向标（投放/增长/带量化指标的运营）；报价分权重高、甲方对"确定性"敏感；预判对手会吹得更响的标（用可核验性反击） |
| 章内写法 | 开篇给基准数据（行业均值 vs 对标值）；每个打法配"预期指标 + 测算依据 + 对标案例"；用表格呈现投入-产出；主动给验收口径与监测机制 |
| 语言温度 | 直接、可量化、敢给口径 |
| 风险禁忌 | 数据必须有来源、案例必须真实（触真实性红线即出局）；预期指标用"目标/力争"措辞，不轻诺"保证" |

### 选择决策（Task 1 判定顺序）

1. **用户显式指定**（命令带 `-logic` / `-story` / `-vision` / `-evidence`）→ 直接采用
2. 否则逐条对照四种策略的"适用信号"，命中最多者为主叙事
3. 势均力敌 → 权重最高的评分维度定夺：技术分主导 → logic/evidence；创意分主导 → story；服务规划/合作机制主导 → vision
4. 四种都不贴 → 允许 `mode: "custom"`，但必须在 `rationale` 写清手法与理由（如"悬念递进式""对比冲击式"）
5. **辅助叙事 `secondary`（可为 null）**：主叙事之外，个别章节可换讲法。**报价章、合规/资质响应类内容永远用 logic/evidence 呈现，不参与故事化**——评委在这两处只想要确定性

> 主叙事全案唯一。一份方案里频繁切换叙事 = 没有叙事。

## 方案输出编号体系

方案正文使用三级编号，章与子节采用不同编号格式以避免层级冲突：

| 层级 | Markdown | 编号格式 | 示例 |
|------|----------|----------|------|
| **章（一级）** | `##` | 汉字数字 + 顿号：一、二、三… | `## 三、传播策略与大概念` |
| **子节（二级）** | `###` | 阿拉伯数字 x.x：1.1, 1.2… | `### 3.1 核心传播主张` |
| **子子节（三级）** | `####` | 括号阿拉伯数字：(1), (2)… | `#### (1) 线上引爆路径` |

### 核心约束

1. **同层统一**：同一层级全篇使用相同编号格式，不混用
2. **不可跳级**：一级 → 二级 → 三级递进
3. **体系分离**：子节用阿拉伯数字，与章的汉字数字分离
4. **每章重置**：子节编号在每章内从 N.1 重新开始
